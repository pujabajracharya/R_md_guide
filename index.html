1) Quick overview: What is R Markdown?

Definition

R Markdown is a file format (.Rmd) that combines formatted text and executable R code. When you Knit, the code runs and the output (tables, figures, numbers) is inserted into a final report.

Key idea

When knitting, R runs from top-to-bottom in a fresh session. Your report should contain everything needed to reproduce the results.

Typical outputs (HTML / PDF / Word)

Choose one output format in the YAML header. For today, use HTML (most reliable).
HTML (recommended)
---
title: "My Report"
output: html_document
---
Word
---
title: "My Report"
output: word_document
---
PDF (often needs LaTeX)
---
title: "My Report"
output: pdf_document
---

Why we care (lab workflows)

Reproducible analyses

Your .Rmd becomes a record of what you did and why, plus the exact code.

Auto-updating figures + tables

Change data or filters → knit again → your report updates consistently.

Collaboration

Share one file containing notes, code, plots, and tables. Easier to review.

Cleaner workflow

Avoid manual copy/paste into Word and reduce errors from mismatched versions.

2) Guided hands-on demo: build a mini report from scratch

We will make a mini report using the built-in dataset mtcars: load packages → small wrangle → one ggplot → one table → knit to HTML.

Create a new .Rmd (from template)

In RStudio: File → New File → R Markdown… → choose HTML → OK → click Knit.
Minimal working Rmd (copy into a new .Rmd file)
---
title: "Mini R Markdown Report"
author: "Your Name"
date: "`r Sys.Date()`"
output: html_document
---

# Introduction
Write your notes here.

```{r}
x <- rnorm(50)
mean(x)
```

YAML header (title/author/date/output)

What YAML does

YAML is the top block between --- lines. It controls the output type and document settings.

HTML with table of contents
---
title: "Mini Report"
author: "Your Name"
date: "`r Sys.Date()`"
output:
  html_document:
    toc: true
    toc_depth: 2
    number_sections: true
---
YAML is indentation-sensitive. If knitting fails immediately, check spacing and --- lines.

Markdown text basics

What Markdown is

Markdown is a simple way to format text using characters. You use it for headings, lists, emphasis, and links.

Markdown formatting examples (paste into your Rmd body)
# Heading 1
## Heading 2

This is **bold**, this is *italic*, and this is `inline code`.

- Bullet 1
- Bullet 2
  - Sub-bullet

A link: https://rmarkdown.rstudio.com/

Code chunks

What a chunk is

A code chunk starts with ```{r} and ends with ```. When you knit, R runs that code and inserts output.

Basic chunk
```{r}
summary(mtcars)
```
Global chunk defaults (put near the top)
```{r setup, include=FALSE}
knitr::opts_chunk$set(
  echo = TRUE,
  message = FALSE,
  warning = FALSE
)
```
Options you will use most
echo=FALSE        # hide code
include=FALSE     # run code but show nothing
message=FALSE     # hide package startup messages
warning=FALSE     # hide warnings
fig.width=7       # plot width
fig.height=4      # plot height
If any chunk errors, knitting stops. Fix the first error shown in the Knit output.

Wrangle + plot + table (mtcars)

Install + load packages (first time only)

Small wrangle + summarise
```{r}
sumdat <- mtcars %>%
  mutate(cyl = factor(cyl)) %>%
  group_by(cyl) %>%
  summarise(
    n = n(),
    mean_mpg = mean(mpg),
    sd_mpg = sd(mpg),
    .groups = "drop"
  )

sumdat
```
One ggplot figure (geom_point)
```{r, fig.width=7, fig.height=4}
raw <- mtcars %>% mutate(cyl = factor(cyl))

ggplot(raw, aes(x = wt, y = mpg, color = cyl)) +
  geom_point(size = 2) +
  theme_bw() +
  labs(
    title = "Fuel efficiency vs weight",
    x = "Weight (1000 lbs)",
    y = "Miles per gallon",
    color = "Cylinders"
  )
```
Small summary table
```{r}
kable(sumdat, digits = 2, caption = "Summary of MPG by cylinder group (mtcars)")
```

Load CSV + set treatment order
```{r}
library(tidyverse)

spring <- read_csv("Comparative_Springtail_AOX_dat.csv")

spring_factors <- spring %>%
  mutate(treat = factor(treat, levels = c("control", "SHAM", "AA")))

levels(spring_factors$treat)
```
Linear model example (PMG.mg ~ treat)
```{r}
mod_1 <- lm(PMG.mg ~ treat, data = spring_factors)
summary(mod_1)
```
geom_point plot (with jitter so points don’t overlap)
```{r, fig.width=7, fig.height=4}
ggplot(spring_factors, aes(x = treat, y = PMG.mg, color = treat)) +
  geom_point(size = 2, position = position_jitter(width = 0.12, height = 0)) +
  theme_bw() +
  labs(title = "PMG.mg by treatment", x = "Treatment", y = "PMG.mg")
```
Optional: points + group mean (mean shown as black dot)
```{r, fig.width=7, fig.height=4}
means <- spring_factors %>%
  group_by(treat) %>%
  summarise(mean_PMG = mean(PMG.mg, na.rm = TRUE), .groups = "drop")

ggplot(spring_factors, aes(x = treat, y = PMG.mg, color = treat)) +
  geom_point(size = 2, position = position_jitter(width = 0.12, height = 0)) +
  geom_point(data = means, aes(x = treat, y = mean_PMG), inherit.aes = FALSE, color = "black", size = 4) +
  theme_bw() +
  labs(title = "PMG.mg by treatment (points + mean)", x = "Treatment", y = "PMG.mg")
```
If CSV loading fails, it’s usually a file path / working directory issue. See “Stuff that breaks.”

Stuff that usually breaks (troubleshooting)

Working directory / file paths

For teaching: keep your .Rmd and your CSV in the same folder. Use relative paths like "data/myfile.csv" if you store data in a folder.

Check where R is looking
```{r}
getwd()
list.files()
```

Missing packages

install.packages() installs once. library() loads every session.

Install vs load
```{r}
install.packages("tidyverse")
library(tidyverse)
```
If knitting fails: fix the first error message. Most common: wrong file path or missing package.
